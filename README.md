## 🍽️ What is RecipeBox?

**RecipeBox** is a modern, full-stack recipe management platform that solves a universal problem — your recipes are everywhere except where you need them. Whether they live in browser tabs, notebook pages, or your memory, RecipeBox brings them all into one beautiful, organized home.

<br/>

<div align="center">

|  | Capability |
|:---:|---|
| 📚 | Organize recipes into **albums** — think of them as curated cookbooks |
| 🤖 | **AI-powered discovery** — type in what's in your fridge, get full recipes |
| 🖼️ | **Image-first design** — every recipe shines with Cloudinary CDN photos |
| 🔐 | **Secure by default** — JWT auth with role-based access control |
| ❤️ | **Personal favorites** — build your own collection of go-to dishes |
| 🛡️ | **Admin dashboard** — full control over users, content, and stats |

</div>

## 🏗️ Architecture

```
╔══════════════════════════════════════════════════════════════╗
║                    🌐  CLIENT BROWSER                        ║
║           React 19 · TypeScript · Vite · Chakra UI           ║
║                                                              ║
║  ┌──────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐  ║
║  │   Home   │ │ Albums │ │Recipes │ │  AI 🤖 │ │ Admin  │  ║
║  └──────────┘ └────────┘ └────────┘ └────────┘ └────────┘  ║
╚═══════════════════════════╦══════════════════════════════════╝
                            ║  REST API over HTTPS
                            ║  Authorization: Bearer <JWT>
                            ▼
╔══════════════════════════════════════════════════════════════╗
║                   ⚙️  ASP.NET CORE 9  API                    ║
║                                                              ║
║  Controllers → Services → Entity Framework Core             ║
║  Auth · Recipes · Albums · Favorites · Users · Images       ║
╚═══╦═══════════════════════════════════════╦════════════════╦═╝
    ║ EF Core                               ║ Cloudinary SDK ║ Groq SDK
    ▼                                       ▼                ▼
┌──────────────┐                  ┌────────────────┐   ┌──────────────┐
│  PostgreSQL  │                  │ Cloudinary CDN │   │  Groq LLaMA  │
│ (Clever Cloud│                  │ Image Storage  │   │   3.3-70b    │
│  managed DB) │                  └────────────────┘   └──────────────┘
└──────────────┘
```

<br/>

**Layer breakdown:**

- **Frontend** — React SPA with protected routes, context-based auth state, and a clean service layer abstracting every API call
- **Backend** — Clean architecture: `Controllers → Services → EF Core → PostgreSQL`. No business logic in controllers
- **Database** — PostgreSQL with code-first EF Core migrations that run automatically on startup
- **Images** — All uploads go to Cloudinary; only the resulting URL is stored in the DB
- **AI** — Groq LLaMA 3.3-70b called client-side for instant, streamed recipe suggestions from ingredients

## 🗄️ Database Schema

<div align="center">

| Table | Primary Key | Description |
|---|:---:|---|
| **`Users`** | `Guid` | Accounts — email, username, password hash, role, block status |
| **`Albums`** | `int` | Recipe categories — name, description, cover image |
| **`Recipes`** | `int` | Core content — name, image, type (`Link`/`Text`), ingredients, instructions |
| **`UserFavorites`** | `UserId + RecipeId` | Junction table — user ↔ saved recipe mapping |

</div>

<br/>

**Entity Relationships:**

```
Users   ──────< Recipes          1 user      → many recipes
Users   ──────< UserFavorites    1 user      → many favorites
Albums  ──────< Recipes          1 album     → many recipes
Recipes ──────< UserFavorites    1 recipe    → many user saves
```

## 🔌 API Reference

> 🔑 `Required` = valid JWT Bearer token &nbsp;|&nbsp; 🛡️ `Admin` = Admin role required

<summary><b>🍲 Recipes</b></summary>
<br/>

## ⚙️ Getting Started

### Prerequisites

<p>
  <img src="https://img.shields.io/badge/Node.js-22+-339933?style=flat-square&logo=nodedotjs&logoColor=white" alt="Node.js 22+"/>
  <img src="https://img.shields.io/badge/.NET_SDK-9.0-512BD4?style=flat-square&logo=dotnet&logoColor=white" alt=".NET 9"/>
  <img src="https://img.shields.io/badge/PostgreSQL-required-4169E1?style=flat-square&logo=postgresql&logoColor=white" alt="PostgreSQL"/>
  <img src="https://img.shields.io/badge/Cloudinary-free_tier-3448C5?style=flat-square&logo=cloudinary&logoColor=white" alt="Cloudinary"/>
  <img src="https://img.shields.io/badge/Groq_API-free_tier-F55036?style=flat-square" alt="Groq API"/>
</p>

<br/>

### Step 1 — Clone

```bash
git clone https://github.com/your-username/RecipeBox.git
cd RecipeBox
```

<br/>

### Step 2 — Configure the Backend

```bash
cd server/RecipeBoxServer
```

Edit `appsettings.Development.json`:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=YOUR_HOST;Port=5432;Database=recipebox;Username=YOUR_USER;Password=YOUR_PASSWORD"
  },
  "Jwt": {
    "Key": "your-super-secret-jwt-key-minimum-32-characters-long"
  },
  "Cloudinary": {
    "CloudName": "your-cloud-name",
    "ApiKey":    "your-api-key",
    "ApiSecret": "your-api-secret"
  }
}
```

Start the API — migrations run automatically on first launch:

```bash
dotnet run
```

```
✅  API  →  http://localhost:6001
📄  Docs →  http://localhost:6001/  (Swagger UI)
```

<br/>

### Step 3 — Configure the Frontend

```bash
cd client
```

Create `.env.local`:

```env
VITE_API_URL=http://localhost:6001
VITE_GROQ_API_KEY=your-groq-api-key-here
```

Install and run:

```bash
npm install
npm run dev
```

```
✅  App  →  http://localhost:3000
```

<br/>

### Step 4 — (Optional) Docker

```bash
# Backend
cd server
docker build -t recipebox-api .
docker run -p 6001:10000 recipebox-api

# Frontend
cd client
docker build -t recipebox-client .
docker run -p 3000:10000 recipebox-client
```
