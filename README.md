## рҹҚҪпёҸ What is RecipeBox?

**RecipeBox** is a modern, full-stack recipe management platform that solves a universal problem вҖ” your recipes are everywhere except where you need them. Whether they live in browser tabs, notebook pages, or your memory, RecipeBox brings them all into one beautiful, organized home.

<br/>

<div align="center">

|  | Capability |
|:---:|---|
| рҹ“ҡ | Organize recipes into **albums** вҖ” think of them as curated cookbooks |
| рҹӨ– | **AI-powered discovery** вҖ” type in what's in your fridge, get full recipes |
| рҹ–јпёҸ | **Image-first design** вҖ” every recipe shines with Cloudinary CDN photos |
| рҹ”җ | **Secure by default** вҖ” JWT auth with role-based access control |
| вқӨпёҸ | **Personal favorites** вҖ” build your own collection of go-to dishes |
| рҹӣЎпёҸ | **Admin dashboard** вҖ” full control over users, content, and stats |

</div>

**Layer breakdown:**

- **Frontend** вҖ” React SPA with protected routes, context-based auth state, and a clean service layer abstracting every API call
- **Backend** вҖ” Clean architecture: `Controllers вҶ’ Services вҶ’ EF Core вҶ’ PostgreSQL`. No business logic in controllers
- **Database** вҖ” PostgreSQL with code-first EF Core migrations that run automatically on startup
- **Images** вҖ” All uploads go to Cloudinary; only the resulting URL is stored in the DB
- **AI** вҖ” Groq LLaMA 3.3-70b called client-side for instant, streamed recipe suggestions from ingredients

## рҹ—„пёҸ Database Schema

<div align="center">

| Table | Primary Key | Description |
|---|:---:|---|
| **`Users`** | `Guid` | Accounts вҖ” email, username, password hash, role, block status |
| **`Albums`** | `int` | Recipe categories вҖ” name, description, cover image |
| **`Recipes`** | `int` | Core content вҖ” name, image, type (`Link`/`Text`), ingredients, instructions |
| **`UserFavorites`** | `UserId + RecipeId` | Junction table вҖ” user вҶ” saved recipe mapping |

</div>

<br/>

**Entity Relationships:**

```
Users   в”Җв”Җв”Җв”Җв”Җв”Җ< Recipes          1 user      вҶ’ many recipes
Users   в”Җв”Җв”Җв”Җв”Җв”Җ< UserFavorites    1 user      вҶ’ many favorites
Albums  в”Җв”Җв”Җв”Җв”Җв”Җ< Recipes          1 album     вҶ’ many recipes
Recipes в”Җв”Җв”Җв”Җв”Җв”Җ< UserFavorites    1 recipe    вҶ’ many user saves
```

## вҡҷпёҸ Getting Started

### Prerequisites

<p>
  <img src="https://img.shields.io/badge/Node.js-22+-339933?style=flat-square&logo=nodedotjs&logoColor=white" alt="Node.js 22+"/>
  <img src="https://img.shields.io/badge/.NET_SDK-9.0-512BD4?style=flat-square&logo=dotnet&logoColor=white" alt=".NET 9"/>
  <img src="https://img.shields.io/badge/PostgreSQL-required-4169E1?style=flat-square&logo=postgresql&logoColor=white" alt="PostgreSQL"/>
  <img src="https://img.shields.io/badge/Cloudinary-free_tier-3448C5?style=flat-square&logo=cloudinary&logoColor=white" alt="Cloudinary"/>
  <img src="https://img.shields.io/badge/Groq_API-free_tier-F55036?style=flat-square" alt="Groq API"/>
</p>

<br/>

### Step 1 вҖ” Clone

```bash
git clone https://github.com/your-username/RecipeBox.git
cd RecipeBox
```

<br/>

### Step 2 вҖ” Configure the Backend

```bash
cd server/RecipeBoxServer
```

Edit `appsettings.Development.json`:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=YOUR_HOST;Port=5432;Database=recipebox;Username=YOUR_USER;Password=YOUR_PASSWORD"
  },
  "Jwt": {
    "Key": "your-super-secret-jwt-key-minimum-32-characters-long"
  },
  "Cloudinary": {
    "CloudName": "your-cloud-name",
    "ApiKey":    "your-api-key",
    "ApiSecret": "your-api-secret"
  }
}
```

Start the API вҖ” migrations run automatically on first launch:

```bash
dotnet run
```

```
вң…  API  вҶ’  http://localhost:6001
рҹ“„  Docs вҶ’  http://localhost:6001/  (Swagger UI)
```

<br/>

### Step 3 вҖ” Configure the Frontend

```bash
cd client
```

Create `.env.local`:

```env
VITE_API_URL=http://localhost:6001
VITE_GROQ_API_KEY=your-groq-api-key-here
```

Install and run:

```bash
npm install
npm run dev
```

```
вң…  App  вҶ’  http://localhost:3000
```

<br/>

### Step 4 вҖ” (Optional) Docker

```bash
# Backend
cd server
docker build -t recipebox-api .
docker run -p 6001:10000 recipebox-api

# Frontend
cd client
docker build -t recipebox-client .
docker run -p 3000:10000 recipebox-client
```
