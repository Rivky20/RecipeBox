## рЯНљпЄП What is RecipeBox?

**RecipeBox** is a modern, full-stack recipe management platform that solves a universal problem вАФ your recipes are everywhere except where you need them. Whether they live in browser tabs, notebook pages, or your memory, RecipeBox brings them all into one beautiful, organized home.

<br/>

<div align="center">

|  | Capability |
|:---:|---|
| рЯУЪ | Organize recipes into **albums** вАФ think of them as curated cookbooks |
| рЯ§Ц | **AI-powered discovery** вАФ type in what's in your fridge, get full recipes |
| рЯЦЉпЄП | **Image-first design** вАФ every recipe shines with Cloudinary CDN photos |
| рЯФР | **Secure by default** вАФ JWT auth with role-based access control |
| вЭ§пЄП | **Personal favorites** вАФ build your own collection of go-to dishes |
| рЯЫ°пЄП | **Admin dashboard** вАФ full control over users, content, and stats |

**Layer breakdown:**

- **Frontend** вАФ React SPA with protected routes, context-based auth state, and a clean service layer abstracting every API call
- **Backend** вАФ Clean architecture: `Controllers вЖТ Services вЖТ EF Core вЖТ PostgreSQL`. No business logic in controllers
- **Database** вАФ PostgreSQL with code-first EF Core migrations that run automatically on startup
- **Images** вАФ All uploads go to Cloudinary; only the resulting URL is stored in the DB
- **AI** вАФ Groq LLaMA 3.3-70b called client-side for instant, streamed recipe suggestions from ingredients

## рЯЧДпЄП Database Schema

<div align="center">

## вЪЩпЄП Getting Started

### Prerequisites

<p>
  <img src="https://img.shields.io/badge/Node.js-22+-339933?style=flat-square&logo=nodedotjs&logoColor=white" alt="Node.js 22+"/>
  <img src="https://img.shields.io/badge/.NET_SDK-9.0-512BD4?style=flat-square&logo=dotnet&logoColor=white" alt=".NET 9"/>
  <img src="https://img.shields.io/badge/PostgreSQL-required-4169E1?style=flat-square&logo=postgresql&logoColor=white" alt="PostgreSQL"/>
  <img src="https://img.shields.io/badge/Cloudinary-free_tier-3448C5?style=flat-square&logo=cloudinary&logoColor=white" alt="Cloudinary"/>
  <img src="https://img.shields.io/badge/Groq_API-free_tier-F55036?style=flat-square" alt="Groq API"/>
</p>

<br/>

### Step 1 вАФ Clone

```bash
git clone https://github.com/your-username/RecipeBox.git
cd RecipeBox
```

<br/>

### Step 2 вАФ Configure the Backend

```bash
cd server/RecipeBoxServer
```

Edit `appsettings.Development.json`:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=YOUR_HOST;Port=5432;Database=recipebox;Username=YOUR_USER;Password=YOUR_PASSWORD"
  },
  "Jwt": {
    "Key": "your-super-secret-jwt-key-minimum-32-characters-long"
  },
  "Cloudinary": {
    "CloudName": "your-cloud-name",
    "ApiKey":    "your-api-key",
    "ApiSecret": "your-api-secret"
  }
}
```

Start the API вАФ migrations run automatically on first launch:

```bash
dotnet run
```

```
вЬЕ  API  вЖТ  http://localhost:6001
рЯУД  Docs вЖТ  http://localhost:6001/  (Swagger UI)
```

<br/>

### Step 3 вАФ Configure the Frontend

```bash
cd client
```

Create `.env.local`:

```env
VITE_API_URL=http://localhost:6001
VITE_GROQ_API_KEY=your-groq-api-key-here
```

Install and run:

```bash
npm install
npm run dev
```

```
вЬЕ  App  вЖТ  http://localhost:3000
```

<br/>

### Step 4 вАФ (Optional) Docker

```bash
# Backend
cd server
docker build -t recipebox-api .
docker run -p 6001:10000 recipebox-api

# Frontend
cd client
docker build -t recipebox-client .
docker run -p 3000:10000 recipebox-client
```
